## 📖 项目概述

BLE 深度扫描器（BLE Deep Scanner）是一款基于 Python + `bleak` 的跨平台 BLE 广播包分析与设备识别工具。它能够实时捕获周围 BLE 设备的所有广播包，并对其进行**多维度深度解析**，涵盖 MAC 地址类型识别、厂商数据库匹配、协议深度识别、信号质量分析、AD 结构完整解析等核心能力。

v10.0 是项目迄今为止**功能最丰富、体系最完整**的版本，在 v7.0 的基础上新增了 **RSSI 实时追踪图表**、**PCAP 导出**、**设备分类统计面板**、**蓝牙适配器信息**、**BLE Mesh 支持**和 **MTU 协商**等六大核心功能，同时大幅扩展了协议解析的覆盖范围。

## 🎯 完整功能特性

### 📡 扫描功能

| 功能 | 说明 |
|------|------|
| 主动/被动扫描模式 | 支持 BLE 主动扫描（可发送扫描请求）和被动扫描（仅侦听广播）两种模式 |
| RSSI 过滤 | 可设置 RSSI 范围（默认 -100 ~ 0 dBm），过滤掉信号过弱的设备 |
| 实时数据更新 | 扫描结果实时更新显示，新设备自动添加，已有设备更新最新广播包 |
| 设备去重 | 自动按 MAC 地址去重，同一设备仅保留最新记录 |
| 目标设备过滤 | 支持按 MAC 地址或设备名称关键词（逗号分隔）精确捕获指定设备 |

### 📊 信号质量分析

| 指标 | 说明 |
|------|------|
| 当前 RSSI | 实时接收信号强度 |
| 平均 RSSI | 历史样本的平均值 |
| 最小 RSSI | 历史样本的最低值 |
| 最大 RSSI | 历史样本的最高值 |
| 样本数量 | 已捕获的广播包数量 |
| 估算距离 | 基于 RSSI 和 Tx Power 的物理距离估算 |

### 🗂 厂商数据库支持

支持加载并合并以下 5 种格式的 OUI 数据库：

| 文件名 | 类型 | 说明 |
|--------|------|------|
| `oui.txt` | IEEE 官方 OUI 文件 | 标准 24 位 OUI 列表 |
| `mam.csv` | MA‑M 分配文件 | 28 位前缀 |
| `oui36.csv` | MA‑S 分配文件 | 36 位前缀 |
| `cid.csv` | CID 分配文件 | 公司 ID |
| `mac_vendors_database.json` | JSON 数组 | 键名为 `mac_prefix` 和 `vendor_name` |

> 将以上任意文件放入同一文件夹，通过 GUI 的「选择 OUI 文件夹」按钮即可一次性加载并合并。加载后可通过「重解析 MAC 厂商」按钮一键更新表格中所有设备的厂商信息。

## 📦 安装与运行

### 系统要求

- **操作系统**：Windows / Linux / macOS（需支持 BLE 蓝牙）
- **Python 版本**：≥ 3.8
- **蓝牙适配器**：内置或外接 BLE 4.0+ 适配器

### 安装步骤

```bash
# 1. 克隆仓库
git clone https://github.com/t84RT/BLE-Deep-Scanner.git
cd BLE-Deep-Scanner

# 2. 安装依赖
pip install bleak psutil

# 3. 运行程序
python RUNtudo/scanV10.0.py
```

### 依赖说明

```
bleak>=0.21.0   # 跨平台 BLE 通信库
psutil>=5.9.0   # 系统资源监控
```
